# AI Agent 经济学与 ToB 商业化定价

## 一、AI Agent 经济学

### AI Agent 的「微观经济学」特性
| 概念 | 传统 SaaS | AI Agent 时代 |
|------|-----------|---------------|
| **边际成本** | 趋近于零（复制软件无成本） | 每次推理消耗 GPU 算力，边际成本 > 0 |
| **定价锚点** | 用户数 / 功能层级 | Token 消耗 / 任务复杂度 / Agent 数量 |
| **规模效应** | 用户越多利润率越高 | 用户越多推理成本越高，利润模型不同 |
| **供需曲线** | 复制的边际成本≈0，供给曲线扁平 | 供给受限于算力，供给曲线陡峭 |

### 核心错位：用户购买的是任务完成，厂商计量的是计算过程
- 现有 API 定价按 Token 计量，但用户价值在于任务完成（如「生成一个营销视频」）
- 错位导致：用户感知不到成本构成，厂商难以传递价值
- 解决方案：按**任务/产出定价** + **Token 成本内部消化**

## 二、AI 产品的计费模式

[来源：a16z《How to Price AI Agents》; Bessemer Venture Partners]

### 五种主流计费模式

| 模式 | 适用场景 | 典型定价 | 优点 | 风险 |
|------|---------|---------|------|------|
| **订阅制（Seat-based）** | 高频固定工作流 | $20-200/用户/月 | 收入可预测 | Agent 替代人力后用户缩量 |
| **消费制（Token/Usage）** | 波动型需求 | $0.01-0.15/次调用 | 按量付费，门槛低 | 成本不可预测，用户抵制 |
| **混合制（Subscription + Credit）** | 多数 AI 产品 | $20/月+$0.01/额外调用 | 基础稳定 + 弹性 | 计费复杂 |
| **成果制（Outcome-based）** | 销售/客服场景 | $10/成功转化 | 与客户价值对齐 | 收入波动大，需反欺诈 |
| **分层容量制（Tiered Capacity）** | 企业客户 | $5K/月（100K Agent 任务） | 容量可预期 | 低估用量侵蚀利润 |

### 关键趋势：从 Seat-based → Consumption-based
- Slack 收购案（$27B）揭示了 seat-based 在 AI 时代的脆弱性——AI Agent 替代人类后座位数骤降
- 趋势：按**价值/成果**定价，而非按「人座」

## 三、ToB 定价的基本概念

### Unit Economics（单位经济模型）
- **CAC（客户获取成本）**：获客总费用 / 新增客户数
- **LTV（客户生命周期价值）**：ARPU × 平均付费周期 × 毛利率
- **Payback Period（回收期）**：CAC / 月度 ARPU，健康值 < 12 个月
- **Churn Rate（流失率）**：月/年流失比例，直接影响 LTV

### 定价策略五步法
1. **锚定价值**：为客户创造的价值是多少？（成本节约/收入增量/效率提升）
2. **竞品对标**：同类产品定价区间
3. **成本底线**：毛利率 > 70%（SaaS 健康基准）
4. **客户支付意愿**：WTP（Willingness to Pay）调研
5. **定价测试**：A/B 测试不同价格点的转化率

## 四、ToB 三种定价模式

[来源：Tom Tunguz《Winning with Data》; 刘铮《SaaS 定价实战》]

### 按价值定价（Value-based Pricing）
- **定义**：根据为客户创造的价值定价
- **例子**：AI 客服工具按「成功解决的工单数」收费
- **适用**：客户价值明确可量化的场景
- **难点**：需要与客户深度共建量化指标

### 按用户/座位定价（Per-seat / Per-user）
- **定义**：按使用人数收费
- **例子**：Notion AI $10/成员/月
- **适用**：协作型工具
- **风险**：AI 提效后企业减少座位，收入下降

### 按使用量定价（Usage-based / Consumption）
- **定义**：按资源消耗量收费
- **例子**：API 调用次数 / Token 消耗
- **适用**：弹性需求客户
- **风险**：客户成本不可控，容易引发「账单冲击」

### 三种部署方式定价矩阵

| 维度 | SaaS | 私有云 | 私有化 |
|------|------|--------|--------|
| **部署成本** | 低（厂商承担） | 中（共享基础设施） | 高（专属硬件） |
| **数据安全** | 标准合规 | 较高 | 最高（本地物理隔离） |
| **定制化** | 有限 | 中等 | 完全定制 |
| **运维责任** | 厂商 | 共担 | 客户/厂商 |
| **适用客户** | 中小企业 | 中型企业 | 大型/政府/金融 |
| **定价模式** | 订阅/按量 | 订阅+资源费 | 项目制+年度维护 |

### 谈判策略（BATNA / ZOPA）
- **BATNA（最佳替代方案）**：谈判破裂后的选择，决定你的底线
- **ZOPA（可接受区间）**：买卖双方的价格交集
- **锚定效应**：先开价者定调，通常先报高价再让步

## 五、客户分层与 LTV/CAC 模型

[来源：Bessemer Venture Partners SaaS 框架；Stripe《AI产品定价》]

### 三层客户模型

| 层级 | 名称 | 定价 | 交付形态 | LTV/CAC 目标 |
|------|------|------|---------|-------------|
| **免费层** | 体验版 | $0 | 基础功能 | 转化率 > 15% |
| **Pro 层** | 专业版 | $9.9-29.9/月 | 多 Agent + 知识库 | LTV/CAC > 3 |
| **Enterprise** | 企业版 | 定制 $5K-50K/年 | 私有化 + 定制 Ontology | 续约率 > 85% |

### 关键公式
- **LTV = ARPU × 平均付费周期 × 毛利率**
- **CAC = 营销费用 + 获客时间成本 / 新增付费用户数**
- **健康标准：LTV/CAC > 3**
- **警戒线：LTV/CAC < 1.5**

## 六、云产品定价（腾讯云岗位专用）

[来源：腾讯云官网；牛客网《一文科普腾讯行业应用岗》]

### 腾讯云 CVM 三种计费模式
| 模式 | 适用场景 | 成本特征 |
|------|---------|---------|
| **包年包月** | 长期稳定业务 | 单价最低，需预付 |
| **按量计费** | 波动/临时业务 | 单价高，随用随付 |
| **抢占式实例** | 容错性高的批处理 | 价格最低，可能被回收 |

### 腾讯 CSIG 岗位匹配点
- **云与智慧事业群（CSIG）**：B端业务核心，涉及商业化、运营、客户成功
- **产品运营型岗位**：推动商业化落地和规模化增长，分析挖掘不同行业客户需求
- **面试钩子**："腾讯云 CVM 三种计费对应不同客户成本敏感度，我在项目定价计算器里实践了类似分层"

## 核心金句
- "订阅制正在退化为流量入口，真正决定格局的是 API 定价、容量调度和 SLA 交付之间的平衡。"
- "用户购买的是任务完成，厂商计量的是计算过程——这是 Agent 时代最大的定价错位。"
- "大模型本质上不是内容分发，而是高成本计算服务，每一次调用都在消耗真实 GPU。"
