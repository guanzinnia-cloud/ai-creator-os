# 中科闻歌 DIP 决策智能平台方法论

## 核心定义
DIP（Decision Intelligence Platform）是**面向企业真实业务场景的决策智能底座**，不是简单 AI 问答工具或传统数据看板。核心是将企业数据、业务逻辑、大模型能力与业务流程深度连接，让 AI 从「会回答问题」升级为「能参与企业决策、推动业务行动」。

## 企业 AI 落地三大痛点（DIP 解决）
1. **数据散，找不到**：分散在 ERP/CRM/OA/邮箱，跨系统整合难
2. **业务复杂，AI 看不懂**：通用大模型无法天然理解企业专属业务逻辑与经营规则
3. **分析完了，行动推不动**：传统工具仅输出结论，无法转化为可执行业务动作

## 核心价值：构建企业「业务孪生」
通过**本体建模（Ontology Modeling）**，将分散数据转化为 AI 可理解的「业务认知网」，复刻真实运转的业务世界。

## 本体建模三步法（DIP 区别于通用 AI 的关键）
1. **数据 → 业务对象**：字段/表格/单据 → 客户、订单、产品、库存、合同、供应商等实体
2. **对象 → 业务关系**：连接客户-订单-库存-合同-履约-回款等关联，让 AI 明确数据间业务关联
3. **关系 → 智能判断**：AI 基于业务认知网，沿关系链查事实、找关联、辨风险，输出可信分析与决策建议

## 四步闭环：从数据到行动
连接数据 → 构建业务认知网 → 智能分析与判断 → 推动业务行动

## 与同类产品的核心差异

| 对比对象 | 通用大模型 | 传统 BI | 知识库/RAG | DIP |
|---------|-----------|---------|-----------|-----|
| 核心能力 | 通用问答 | 数据展示 | 文档检索归纳 | 理解业务、分析原因、推动行动 |
| 解决问题 | 通用知识 | 发生了什么 | 文档里有什么 | 为什么发生？风险在哪？下一步怎么办？ |
| 数据关联 | 无 | 单系统 | 仅文本 | 多系统数据+规则+流程打通 |
| 行动闭环 | 无 | 无 | 无 | 转化为任务/提醒/审批流 |

## 核心落地场景
1. **客户流失预警**：识别潜在流失客户 → 归因 → 推送销售跟进任务
2. **经营分析报告**：自动汇总跨部门数据 → 含分析/判断/建议的报告
3. **合同风险审核**：条款比对 + 关联历史履约/客户等级/回款 → 风险提示
4. **库存补货决策**：销售趋势 + 安全库存 + 供应商履约 → 补货量 + 跟进人
5. **周期性业务监控**：定期生成全景报告，持续监控异常，自动预警推送负责人

## 真实案例：中源生物
- **背景**：100万+ SKU，数万客户，业务规模大、产品复杂、系统分散
- **落地**：
  - 经营分析：跨部门报告 + 客户/供应商风险 + 库存健康度 + 资金进度
  - 合同审核：条款比对 + 历史履约数据 → 风险判断
  - 补货决策：多维度数据 → 精准补货方案 → 推送执行流程

## 关键特性
1. **主动发现问题**：周期性任务运行，持续监控，无需人工提问即可自动预警
2. **兼容现有系统**：从单一高频场景切入逐步扩展，复用已有数据与能力
3. **灵活部署定价**：SaaS/私有化/私有云，按业务复杂度与实际需求灵活定价

## 与 AI Creator OS 的映射关系
| DIP 概念 | AI Creator OS 映射 | 说明 |
|---------|-------------------|------|
| 本体建模（业务认知网） | **Ontology 层**（DOMA） | 博主定位体系、用户画像本体、内容分类体系 |
| 业务对象 | 细分用户群、内容类型、平台规则 | 结构化实体定义 |
| 业务关系 | 用户-痛点-内容-平台的关联图谱 | 知识库 Agent 的关联逻辑 |
| 智能判断 | 6 个 Agent 的协作决策 | 基于本体层的推理与执行 |
| 推动行动 | 内容生产流水线 + 飞书数据回流 | 分析 → 脚本 → 视频 → 运营 → 变现 |

## 核心金句
- "没有本体，大模型就不懂业务"（王磊，DOMA）
- "DIP 的核心价值是打破企业数据孤岛与 AI 业务理解壁垒"
- "从人工汇总+经验判断 → 业务建模+智能决策"
- "让 AI 真正进入业务流程、辅助决策、推动行动"
